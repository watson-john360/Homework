Introduction to Phabricator

Uber ATG currently uses a suite of development tools called Phabricator (website, Github) for software development. These tools include:

  • Repository tool (Diffusion)
  • Review tool (Differential)
  • Code auditing tool (Herald)
  • Project management/bug tracking tool (Maniphest)
  • Workboard (aptly named Workboards)
  • Chat channel (Conpherence)
  • Documentation wiki (Phriction)

Notably, it also contains a command line tool (arcanist) which grants CLI access to (most) of Phabricator's functionality, and a HTTP JSON API for application programming (Conduit API). See Landing Code with Phabricator for more information on how Uber ATG uses arcanist and Conduit.

So why Phabricator?

Phabricator offers a few key advantages, with the main one being all of the tools are designed to work together seamlessly. A developer can clone a repository using Diffusions, write code, and submit it for review using arcanist. The code can then be reviewed using Differential, issues can be tracked using Maniphest, discussions can be had over Conpherence, and code audits can be automatically triggered using Herald, etc. Since there are no plug-ins or third-party scripts to worry about, the entire ATG organization work from one unified system.
